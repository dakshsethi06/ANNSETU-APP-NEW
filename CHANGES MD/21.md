# Mandi Price API Caching & Pagination Report

This document details the changes made to cache external government Mandi price API results and implement pagination on the backend.

---

## 1. Summary of Changes

1. **In-Memory TTL Caching**: Government Mandi price API fetches are slow, rate-limited, and change at most daily. We implemented an in-memory Map cache on the backend with a **1-hour TTL** (Time to Live) to cache query results keyed by `state:commodity:market`.
2. **Paginated Responses**: Added `page` and `limit` query parameters on the `GET /api/mandi` endpoint, allowing sliced records to be returned with a standard pagination metadata envelope (`page`, `limit`, `totalPages`, `totalRecords`).

const CACHE_TTL_MS = 60 * 60 * 1000; // 1 hour

function getCacheKey(state, commodity, market) {
  return `${state || 'All'}:${commodity || 'All'}:${market || 'All'}`;
}

async function fetchMandiPrices(apiKey, state, commodity, market) {
  const cacheKey = getCacheKey(state, commodity, market);
  const cachedEntry = cache.get(cacheKey);

  // Check cache hit

    console.log(`[Mandi Cache] Cache hit for key: ${cacheKey}`);

const limit = parseInt(req.query.limit, 10) || 10;

// Slice records array

return res.json({
  success: true,
